# 企业支持

Tairitsu 为部署生产应用的组织提供企业级支持。

## 支持级别

### 社区支持（免费）

- 公开文档和指南
- 社区 Discord 和 GitHub Discussions
- 通过公共发布修复 Bug
- 最佳实践文档

### 专业支持

需要专门支持的组织请联系我们：

- 优先 Bug 修复和安全补丁
- 直接访问核心开发者
- 定制集成协助
- 架构审查和指导
- SLA 支持的响应时间

### 企业支持

企业许可证包括：

- 24/7 关键问题支持
- 专职支持工程师
- 现场培训和研讨会
- 定制功能开发
- 长期支持 (LTS) 版本
- 安全审计协助

## SLA（服务级别协议）

### 专业支持

| 严重性 | 响应时间 | 解决时间 |
|--------|----------|----------|
| 关键 (P1) | 4 小时 | 2 个工作日 |
| 高 (P2) | 1 个工作日 | 5 个工作日 |
| 中 (P3) | 2 个工作日 | 2 周 |
| 低 (P4) | 1 周 | 尽力而为 |

### 企业支持

| 严重性 | 响应时间 | 解决时间 |
|--------|----------|----------|
| 关键 (P1) | 1 小时 | 24 小时 |
| 高 (P2) | 4 小时 | 2 个工作日 |
| 中 (P3) | 1 个工作日 | 1 周 |
| 低 (P4) | 2 个工作日 | 2 周 |

## 严重性定义

### P1 - 关键
- 生产系统完全停机
- 数据丢失或损坏
- 存在主动利用的安全漏洞

### P2 - 高
- 主要功能无法工作
- 显著性能下降
- 有变通方法但影响生产力

### P3 - 中
- 次要功能问题
- 需要不方便的变通方法
- 文档错误

### P4 - 低
- 外观问题
- 功能请求
- 一般问题

## 长期支持 (LTS)

LTS 版本提供：

- 2 年的 Bug 修复
- 3 年的安全更新
- 关键问题的回移植
- 稳定的 API 保证

当前 LTS 版本：无（首个 LTS：v1.0.0）
