## 报告漏洞

如果您发现安全漏洞，请负责任地报告。

### 如何报告

**邮箱**：security@tairitsu.dev

请包含：
- 漏洞描述
- 重现步骤
- 潜在影响
- 概念验证（如有）

### 预期结果

1. **确认**：我们会在 24 小时内确认收到
2. **评估**：我们会在 3 个工作日内调查并评估严重性
3. **解决**：我们将开发并测试修复
4. **披露**：我们将与您协调披露

### 披露策略

- 关键/高严重性：7 天内修复，修复部署后披露
- 中严重性：30 天内修复，修复部署后披露
- 低严重性：在下个版本中修复，公开披露

## 安全最佳实践

### 应用开发者

1. **保持依赖更新**
   ```bash
   cargo update
   ```

2. **启用内容安全策略**
   ```toml
   [packager.security]
   csp = "default-src 'self'; script-src 'self' 'unsafe-inline'"
   ```

3. **清理用户输入**
   ```rust
   use tairitsu_web::sanitize;

   let safe_html = sanitize_html(user_input);
   ```

4. **生产环境使用 HTTPS**
   ```toml
   [packager.server]
   force_https = true
   ```

### 部署

1. **使用验证的容器镜像**
2. **启用子资源完整性 (SRI)**
3. **实施速率限制**
4. **定期安全审计**

## 安全功能

### 内置保护

- **XSS 防护**：自动 HTML 转义
- **CSP 支持**：内容安全策略标头
- **SRI**：外部资源的子资源完整性
- **安全 SVG**：清理过的 SVG 嵌入

### WASM 沙箱

Tairitsu 在浏览器沙箱中运行 WebAssembly：
- 内存隔离
- 无直接 DOM 访问（通过 WIT）
- 基于能力的安全性
