# セキュリティポリシー

## サポート対象バージョン

以下のバージョンにセキュリティ更新を提供：

| バージョン | ステータス     | セキュリティ更新 |
|------------|---------------|------------------|
| 0.3.x      | 最新          | はい             |
| 0.2.x      | メンテナンス  | はい（重要のみ） |
| < 0.2.0    | サポート終了  | いいえ           |

## 脆弱性の報告

セキュリティ脆弱性を発見した場合は、責任ある開示をお願いします。

### 報告方法

**メール**：security@tairitsu.dev

以下を含めてください：
- 脆弱性の説明
- 再現手順
- 潜在的な影響
- 概念実証（利用可能な場合）

### 期待される対応

1. **確認**：24時間以内に受領を確認いたします
2. **評価**：3営業日以内に調査と重要度評価を行います
3. **解決**：修正の開発とテストを行います
4. **開示**：調整して開示を行います

### 開示ポリシー

- 重要/高重要度：7日以内に修正、修正展開後に開示
- 中重要度：30日以内に修正、修正展開後に開示
- 低重要度：次回リリースで修正、公開開示

## セキュリティベストプラクティス

### アプリケーション開発者向け

1. **依存関係の更新**
   ```bash
   cargo update
   ```

2. **コンテンツセキュリティポリシーの有効化**
   ```toml
   [packager.security]
   csp = "default-src 'self'; script-src 'self' 'unsafe-inline'"
   ```

3. **ユーザー入力のサニタイズ**
   ```rust
   use tairitsu_web::sanitize;

   let safe_html = sanitize_html(user_input);
   ```

4. **本番環境でのHTTPS使用**
   ```toml
   [packager.server]
   force_https = true
   ```

### デプロイ向け

1. **検証済みコンテナイメージの使用**
2. **サブリソース完全性（SRI）の有効化**
3. **レート制限の実装**
4. **定期的なセキュリティ監査**

## セキュリティ機能

### 組み込み保護

- **XSS防止**：自動HTMLエスケープ
- **CSPサポート**：コンテンツセキュリティポリシーヘッダー
- **SRI**：外部リソースのサブリソース完全性
- **安全なSVG**：サニタイズされたSVG埋め込み

### WASMサンドボックス

TairitsuはブラウザサンドボックスでWebAssemblyを実行：
- メモリ分離
- WIT経由の直接DOMアクセスなし
- ケーパビリティベースのセキュリティ
